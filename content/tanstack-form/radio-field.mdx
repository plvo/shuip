## Standalone radio group integration

The simple radio field pattern provides a straightforward way to implement radio button groups with TanStack Form. Each field receives the form instance as a prop, offering excellent type inference and explicit data flow.

## Core characteristics

- **Zero configuration**: No context setup required
- **Type-safe field names**: Autocomplete for nested paths via `DeepKeys<TFormData>`
- **Array-based options**: Simple string array for radio choices
- **Explicit dependencies**: Form instance passed directly to each field
- **Automatic layout**: Vertical stacking of radio items with proper spacing

## Basic usage

<CodeHighlight code={`import { useForm } from '@tanstack/react-form'
import { RadioField } from '@/components/ui/tanstack-form/radio-field'

function SubscriptionForm() {
  const form = useForm({
    defaultValues: {
      plan: '',
    },
    onSubmit: async ({ value }) => {
      await selectPlan(value.plan)
    },
  })

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault()
        form.handleSubmit()
      }}
    >
      <RadioField
        form={form}
        name='plan'
        options={['Free', 'Pro', 'Enterprise']}
        label='Subscription Plan'
        description='Choose your subscription tier'
      />
      <button type='submit'>Continue</button>
    </form>
  )
}`} language='tsx' />

## Type inference

The component provides full type safety for radio selections:

<CodeHighlight code={`type FormData = {
  plan: 'free' | 'pro' | 'enterprise'
  billing: 'monthly' | 'yearly'
}

const form = useForm<FormData>({
  defaultValues: {
    plan: 'free',
    billing: 'monthly',
  }
})

// TypeScript ensures 'plan' is a valid field name
<RadioField
  form={form}
  name='plan'
  options={['free', 'pro', 'enterprise']}
  label='Plan'
/>`} language='tsx' />

The `name` prop is constrained to `DeepKeys<TFormData>`, ensuring only valid field paths are accepted.

## Validation integration

TanStack Form supports validation on radio field selections:

<CodeHighlight code={`const form = useForm({
  defaultValues: { plan: '' },
})

<form.Field
  name='plan'
  validators={{
    // Require a selection
    onChange: ({ value }) =>
      !value ? 'Please select a plan' : undefined,

    // Custom validation
    onBlur: ({ value }) =>
      value === 'enterprise' && !isPremiumUser()
        ? 'Enterprise plan requires premium account'
        : undefined,
  }}
>
  {(field) => (
    <RadioField
      form={form}
      name='plan'
      options={['Free', 'Pro', 'Enterprise']}
      label='Subscription Plan'
    />
  )}
</form.Field>`} language='tsx' />

The component automatically displays validation errors:

## Conditional options

Options can be computed dynamically:

<CodeHighlight code={`function PricingForm() {
  const isPremium = usePremiumStatus()

  const plans = isPremium
    ? ['Free', 'Pro', 'Enterprise', 'Ultimate']
    : ['Free', 'Pro']

  return (
    <RadioField
      form={form}
      name='plan'
      options={plans}
      label='Available Plans'
    />
  )
}`} language='tsx' />

## Accessibility

The component implements proper accessibility patterns:

- Each radio item has a unique `id` based on field name and value
- Labels are properly associated with radio inputs via `htmlFor`
- ARIA attributes for invalid state
- Keyboard navigation support (arrow keys to navigate between options)

## When to use this pattern

**Ideal for:**
- Simple forms with few fields
- Quick prototypes
- Single-choice selections from 2-5 options
- Forms without complex composition needs

**Consider the composed pattern when:**
- Building multiple forms across your app
- Creating reusable form component libraries
- Need lazy loading for large forms

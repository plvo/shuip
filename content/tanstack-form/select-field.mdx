## Standalone select field integration

The simple select field pattern provides a straightforward way to implement dropdown selections with TanStack Form. Each field receives the form instance as a prop, offering excellent type inference and explicit data flow.

## Core characteristics

- **Zero configuration**: No context setup required
- **Type-safe field names**: Autocomplete for nested paths via `DeepKeys<TFormData>`
- **Object-based options**: Key-value pairs where key is label, value is the stored value
- **Explicit dependencies**: Form instance passed directly to each field
- **Radix UI integration**: Built on top of Radix UI Select for accessibility

## Basic usage

<CodeHighlight code={`import { useForm } from '@tanstack/react-form'
import { SelectField } from '@/components/ui/tanstack-form/select-field'

function CountryForm() {
  const form = useForm({
    defaultValues: {
      country: '',
    },
    onSubmit: async ({ value }) => {
      await saveCountry(value.country)
    },
  })

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault()
        form.handleSubmit()
      }}
    >
      <SelectField
        form={form}
        name='country'
        options={{
          'United States': 'us',
          'United Kingdom': 'uk',
          'France': 'fr',
          'Germany': 'de',
        }}
        label='Country'
        placeholder='Select a country'
        description='Choose your country of residence'
      />
      <button type='submit'>Continue</button>
    </form>
  )
}`} language='tsx' />

## Type inference

The component provides full type safety for select values:

<CodeHighlight code={`type FormData = {
  priority: 'low' | 'medium' | 'high'
  status: 'pending' | 'active' | 'completed'
}

const form = useForm<FormData>({
  defaultValues: {
    priority: 'medium',
    status: 'pending',
  }
})

// TypeScript ensures 'priority' is a valid field name
<SelectField
  form={form}
  name='priority'
  options={{
    'Low Priority': 'low',
    'Medium Priority': 'medium',
    'High Priority': 'high',
  }}
  label='Priority Level'
/>`} language='tsx' />

The `name` prop is constrained to `DeepKeys<TFormData>`, ensuring only valid field paths are accepted.

## Validation integration

TanStack Form supports validation on select field values:

<CodeHighlight code={`const form = useForm({
  defaultValues: { country: '' },
})

<form.Field
  name='country'
  validators={{
    // Require a selection
    onChange: ({ value }) =>
      !value ? 'Please select a country' : undefined,

    // Async validation (e.g., check if shipping available)
    onChangeAsync: async ({ value }) => {
      const available = await checkShippingAvailability(value)
      return available ? undefined : 'Shipping not available to this country'
    },
  }}
>
  {(field) => (
    <SelectField
      form={form}
      name='country'
      options={{
        'United States': 'us',
        'Canada': 'ca',
      }}
      label='Shipping Country'
    />
  )}
</form.Field>`} language='tsx' />

## Error handling

The component automatically displays validation errors:

// - field.state.meta.errors: Array of error messages

## Accessibility

The component implements proper accessibility patterns via Radix UI Select:

- Keyboard navigation (arrow keys, Enter, Escape)
- ARIA attributes for screen readers
- Focus management
- Proper labeling via `htmlFor` and `id`
- Invalid state indication

## When to use this pattern

**Ideal for:**
- Simple forms with fewer than 10 fields
- Quick prototypes
- Single-choice selections from 5+ options
- Forms without complex composition needs
- Static or semi-static option lists

**Consider the composed pattern when:**
- Building multiple forms across your app
- Creating reusable form component libraries
- Need lazy loading for large forms
- Want to minimize props on field components

## Type-safe forms with zero boilerplate

<CodeHighlight code={`import { useForm } from '@tanstack/react-form'
import { InputField } from '@/components/ui/shuip/tanstack-form/input-field'
import { SubmitButton } from '@/components/ui/shuip/tanstack-form/submit-button'

const form = useForm({
  defaultValues: { email: '', username: '' },
  onSubmit: async ({ value }) => {
    await saveUser(value)
  }
})

<form onSubmit={(e) => {
  e.preventDefault()
  form.handleSubmit()
}}>
  <InputField form={form} name='email' label='Email' />
  <InputField form={form} name='username' label='Username' />
  <SubmitButton form={form}>Submit</SubmitButton>
</form>`} language='tsx' />

## Built-in features

- **Full type inference**: Autocomplete for field names and values
- **Nested field support**: Deep paths like `user.profile.email`
- **Async validation**: Built-in support for API validation
- **Field metadata**: Access to touched, dirty, validating states
- **Tooltip support**: InfoIcon with contextual help
- **InputGroup integration**: shadcn InputGroup for addons and buttons

## Common validation patterns

<CodeHighlight code={`// Synchronous validation
<InputField
  form={form}
  name='email'
  label='Email'
  formProps={{
    validators: {
      onChange: ({ value }) =>
        !value.includes('@') ? 'Invalid email' : undefined
    }
  }}
/>

// Async validation
<InputField
  form={form}
  name='username'
  label='Username'
  formProps={{
    validators: {
      onChangeAsync: async ({ value }) => {
        const available = await checkAvailability(value)
        return available ? undefined : 'Username taken'
      }
    }
  }}
/>

// Multiple validators
<InputField
  form={form}
  name='password'
  label='Password'
  formProps={{
    validators: {
      onChange: ({ value }) =>
        value.length < 8 ? 'Too short' : undefined,
      onBlur: ({ value }) =>
        !/[A-Z]/.test(value) ? 'Need uppercase' : undefined
    }
  }}
/>`} language='tsx' />

## Why use SelectField?

### Less verbose code in forms

With classic shadcn/ui, creating a select field requires a lot of repetitive code:

<CodeHighlight code={`<FormField
  control={form.control}
  name="country"
  render={({ field }) => (
    <FormItem>
      <FormLabel>Country</FormLabel>
      <Select onValueChange={field.onChange} defaultValue={field.value}>
        <FormControl>
          <SelectTrigger>
            <SelectValue placeholder="Select a country" />
          </SelectTrigger>
        </FormControl>
        <SelectContent>
          <SelectItem value="us">United States</SelectItem>
          <SelectItem value="ca">Canada</SelectItem>
          <SelectItem value="fr">France</SelectItem>
        </SelectContent>
      </Select>
      <FormDescription>
        Choose your country
      </FormDescription>
      <FormMessage />
    </FormItem>
  )}
/>`} language='tsx' />

With `SelectField`, the same result in a single line:

<CodeHighlight code={`<SelectField 
  control={form.control} 
  name="country" 
  label="Country" 
  placeholder="Select a country"
  description="Choose your country"
  values={countryOptions}
/>`} language='tsx' />

### Built-in features

- **Automatic option rendering**: Pass an array of options and they're rendered automatically
- **Type-safe values**: Full TypeScript support for option values
- **Zod validation**: Native integration with react-hook-form and Zod
- **Placeholder support**: Built-in placeholder handling
- **Error display**: Automatic error message display

### react-hook-form integration

The component is specifically designed for react-hook-form:

<CodeHighlight code={`const form = useForm({
      defaultValues: { category: '' },
      resolver: zodResolver(schema),
});

const options = [
      { label: 'Technology', value: 'tech' },
      { label: 'Design', value: 'design' },
      { label: 'Business', value: 'business' },
];

// Direct usage with form control
<SelectField 
      control={form.control} 
      name="category" 
      label="Category" 
      values={options}
/>`} language='tsx' />

## Context

`shuip` is a React component library for Next.js apps, distributed via a custom shadcn registry hosted on `shuip.plvo.dev`. The project has accumulated structural friction:

- A single component spans **5 source locations** (`registry/`, `examples/`, `content/`, `src/components/ui/shuip/`, `registry.json`) вЂ” 5 places to edit/sync per change.
- Three TS path aliases (`@/`, `#/`, `@r/`) with subtle rules; `@r/*` is declared but unused.
- A duplicated `cn` utility (`src/lib/cn.ts` and `src/lib/utils.ts`).
- A 215KB autogenerated barrel (`registry/__index__.ts`) embedding source code as strings.
- Manual maintenance of `registry.json` (item names, deps, targets) вЂ” easy to drift.
- A copy script (`copy-local-registry.ts`) that simulates a "user install" inside `src/components/ui/shuip/` so the docs site can render previews using the same import paths users would write.

Goals:
1. **One component = one folder** вЂ” collocate source, examples, and docs.
2. **Clean import contract** вЂ” at most one alias per role, no surprises.
3. **Doc / lib separation** вЂ” `apps/docs` and `packages/registry` independent.
4. **Easier onboarding** вЂ” explicit mental model.
5. **Zero breaking changes for users** вЂ” public contract (URLs, item `name`, `target`, file content) preserved.

Non-goals:
- Adding tests (no test runner exists today).
- Renaming any registry item (would break user installs).
- Publishing `@repo/ui` to npm.
- Optimizing the `__index__.ts` size вЂ” separate concern.

## TypeScript path resolution

`apps/docs/tsconfig.json`:
```jsonc
{
  "extends": "@repo/config/ts/nextjs.json",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*":                       ["./src/*"],
      "@/components/ui/shuip/*":   ["../../packages/registry/stubs/*"],
      "@/components/ui/*":         ["../../packages/ui/src/components/ui/*"],
      "@/lib/utils":               ["../../packages/ui/src/lib/utils"]
    }
  }
}
```

Order matters: `shuip/*` before generic `ui/*`.

`packages/registry/tsconfig.json` extends `@repo/config/ts/with-shadcn-paths.json` so registry source files type-check standalone (their `@/components/ui/*` imports resolve to `packages/ui` from the registry package's perspective too).

`packages/ui/tsconfig.json` extends `@repo/config/ts/react-library.json` (no special paths).

### Import resolution per consumer

| Consumer | Import | Resolves to |
|---|---|---|
| User app (after shadcn install) | `@/components/ui/form` | their own `components/ui/form.tsx` |
| User app | `@/components/ui/shuip/submit-button` | their own (target installed by shadcn CLI) |
| `apps/docs` MDX | `@/components/ui/shuip/submit-button` | stub в†’ `@repo/registry/items/.../component` |
| `apps/docs` MDX | `@/components/blocks-list` | `apps/docs/src/components/blocks-list.tsx` |
| `apps/docs` MDX | `from 'fumadocs-ui/components/...'` | npm package |
| `apps/docs` `<ItemPreview>` | `@repo/registry/__index__` | `packages/registry/__index__.ts` |
| `packages/registry/items/.../component.tsx` (preview) | `@/components/ui/form` | tsconfig path в†’ `packages/ui/...` |
| `packages/registry/items/.../component.tsx` (shipped) | `@/components/ui/form` | NOT resolved at build вЂ” kept literal in shipped JSON |

## The single generator: `packages/registry/scripts/generate.ts`

Replaces `generate-registry.ts`, `copy-local-registry.ts`, and the hand-edited `registry.json`. Runs on `predev`/`prebuild`. Reads `packages/registry/items/`, writes three artifacts atomically.

### Output 1 вЂ” `packages/registry/registry.json`

Per item, derived from filesystem:
- `name`: prefix-aware. For `items/react-hook-form/<name>/` в†’ `rhf-<name>`. For `items/tanstack-form/<name>/` в†’ `tsf-<name>`. For `items/components/<name>/` and `items/blocks/<name>/` в†’ `<name>`. (Preserves current public contract.)
- `type`: derived from category. `components/` and `react-hook-form/` and `tanstack-form/` в†’ `registry:ui`. `blocks/` в†’ `registry:block`.
- `dependencies`: parsed from imports of `component.tsx`. npm packages only вЂ” exclude `@repo/*` and `@/*`.
- `registryDependencies`: parsed from `@/components/ui/<name>` imports (matches shadcn primitive names). Augmented by `meta.json` `dependsOn` array for cross-shuip deps (e.g. `responsive-dialog` declares `side-dialog`).
- `files[0]`: `{ path, type, target }` where:
  - `path` = `items/<category>/<name>/component.tsx`
  - `target` = computed per category: `./components/ui/shuip/<name>.tsx` for components/blocks, `./components/ui/shuip/react-hook-form/<name>.tsx` for RHF (with `rhf-` stripped), `./components/ui/shuip/tanstack-form/<name>.tsx` for TSF.
- `files[1..]`: each `extras/*` becomes a `registry:file`. Filename pattern `<name>.action.ts` в†’ target `./actions/shuip/<name>.ts`. Other patterns documented as needed.

### Output 2 вЂ” `packages/registry/__index__.ts`

Same shape as today's barrel. Used by `apps/docs/src/components/item-{content,preview}.tsx` for previews. Imports use `@repo/registry/items/...` instead of `#/registry/...`.

### Output 3 вЂ” `packages/registry/stubs/<target-path>.tsx`

One-line re-export per item:
```tsx
// AUTO-GENERATED by scripts/generate.ts вЂ” DO NOT EDIT
export * from '../items/components/submit-button/component';
```

Path mirrors `target` from `registry.json` (without leading `./components/ui/shuip/`). `apps/docs/tsconfig.json` maps `@/components/ui/shuip/*` to this folder, so all MDX and example imports resolve transparently.

Stubs are gitignored at the package level (`packages/registry/.gitignore: stubs/`). Generated on every dev/build run via turbo.

## Build pipeline (turbo)

```jsonc
// turbo.json
{
  "$schema": "https://turborepo.com/schema.json",
  "ui": "stream",
  "tasks": {
    "registry:generate": {
      "inputs": ["items/**", "scripts/generate.ts"],
      "outputs": ["registry.json", "__index__.ts", "stubs/**"]
    },
    "registry:build": {
      "dependsOn": ["@repo/registry#registry:generate"],
      "inputs": ["../../packages/registry/registry.json", "../../packages/registry/items/**"],
      "outputs": ["public/r/**"]
    },
    "build": {
      "dependsOn": ["^build", "registry:generate", "registry:build"],
      "inputs": ["$TURBO_DEFAULT$", ".env*"],
      "outputs": [".next/**", "!.next/cache/**"]
    },
    "dev": {
      "dependsOn": ["registry:generate"],
      "inputs": ["$TURBO_DEFAULT$", ".env"],
      "cache": false,
      "persistent": true
    }
  }
}
```

Sequencing:
- **`bun dev`**: `registry:generate` (cached on `items/**` hash) в†’ `apps/docs` dev (Turbopack picks up `@repo/registry` imports natively).
- **`bun build`**: `registry:generate` в†’ `apps/docs#registry:build` (writes `public/r/`) в†’ `apps/docs#build`.
- fumadocs-mdx watches both `./content/docs` and `../../packages/registry/items` via the `dir` array.

## Naming choices

- `@repo/*` for all internal packages (`@repo/docs`, `@repo/registry`, `@repo/ui`, `@repo/config`) вЂ” matches `create-faster` convention.
- `component.tsx` (not `index.tsx`) вЂ” leaves `index.mdx` free for the doc.
- `*.example.tsx` вЂ” auto-discovered by `generate.ts`, no manual list.

## What disappears

- `registry/`, `examples/`, `content/` at repo root
- `src/` at repo root (everything moves into `apps/docs/src/` or `packages/ui/src/`)
- `src/components/ui/shuip/` (replaced by stubs in `packages/registry/`)
- `src/actions/shuip/` (becomes `extras/places.action.ts` colocated with `address-field`)
- Hand-maintained `registry.json` (now generated)
- `scripts/copy-local-registry.ts`, `scripts/generate-registry.ts` (replaced by single `generate.ts`)
- `cli.json` (fumadocs CLI config, unused)
- `src/lib/cn.ts` (duplicate of `utils.ts`)
- TS aliases `@r/*` (dead) and `#/*` (replaced by `@repo/registry/*`)

## What stays unchanged for users

The shadcn-CLI public contract:
- URLs: `https://shuip.plvo.dev/r/<name>.json`
- Item `name` values (e.g. `submit-button`, `rhf-input-field`)
- `files[].target` install paths
- Shipped file content (literal `@/components/ui/form` imports preserved)

## Migration strategy (progressive)

### Phase 0 вЂ” Skeleton + pilot

1. Create branch `refactor/turborepo`.
2. Add root `turbo.json`, root `package.json` (workspaces + catalogs), root `bunfig.toml`, root `biome.json`.
3. Create `packages/config/` with TS configs.
4. Create `packages/ui/` (empty primitives, `lib/utils.ts`, `styles/globals.css`, `components.json`).
5. Create `packages/registry/` (empty `items/`, `scripts/generate.ts`, `package.json` with `exports`).
6. Create `apps/docs/` (Next.js + fumadocs scaffold, `tsconfig.json`, `next.config.ts`, `source.config.ts`).
7. Migrate ONE pilot: `submit-button` (component + index.mdx + 2 examples).
8. `bun install`, `bun dev`, verify preview at `/docs/components/submit-button`.
9. `bun build`, verify `apps/docs/public/r/submit-button.json`.
10. Smoke test: `npx shadcn add http://localhost:3000/r/submit-button.json` against a throwaway Next.js project.

**Gate**: pilot must work in dev AND build AND `shadcn add` install before continuing.

### Phase 1 вЂ” Move primitives

1. Move `src/components/ui/*.tsx` (excluding `shuip/`) в†’ `packages/ui/src/components/ui/`.
2. Move `src/lib/utils.ts` в†’ `packages/ui/src/lib/utils.ts`.
3. Delete `src/lib/cn.ts` (duplicate).
4. Move `src/styles/globals.css` в†’ `packages/ui/src/styles/globals.css`.
5. Move `components.json` в†’ `packages/ui/components.json` with updated aliases.
6. Wire `apps/docs/app/layout.tsx` to import `@repo/ui/styles/globals.css`.
7. Verify pilot still works.

### Phase 2 вЂ” Migrate items in batches

Batches in order of blast radius (smallest first):

| # | Items |
|---|---|
| 2.1 | `components/`: copy-button, theme-button, confirmation-dialog, hover-reveal, side-dialog |
| 2.2 | `react-hook-form/`: input-field, password-field, checkbox-field, radio-field, select-field, textarea-field, query-boundary, address-field |
| 2.3 | `tanstack-form/`: 7 items (input, password, checkbox, radio, select, textarea, submit-button) |
| 2.4 | `blocks/`: title-section, responsive-dialog (declares `dependsOn: ["side-dialog"]`) |

Per item:
1. Create `packages/registry/items/<category>/<name>/`.
2. Move source: `registry/ui/<file>.tsx` в†’ `items/<category>/<name>/component.tsx`.
3. Move examples: `examples/<file>*.tsx` в†’ `<variant>.example.tsx` (rename `<name>.tsx` в†’ `default.example.tsx`).
4. Move doc: `content/docs/<category>/<name>.mdx` в†’ `items/<category>/<name>/index.mdx`.
5. Verify in browser.
6. Commit per batch (not per item).

After each batch: regenerate `registry.json`, diff against pre-migration version, confirm only the batch's items changed and `name`/`target`/`dependencies` are stable.

### Phase 3 вЂ” Move non-component pages

1. Move `content/docs/{changelog, configuration, contribution, installation, index}.mdx` в†’ `apps/docs/content/docs/`.
2. Move `content/docs/meta.json` в†’ `apps/docs/content/docs/meta.json`.
3. Move `content/blocks/index.mdx` and verify `/blocks` route still works (or merge into `/docs/blocks/...`).
4. Confirm `dir: [...]` resolves both correctly.

### Phase 4 вЂ” Cleanup

1. Delete root `registry/`, `examples/`, `content/`, `src/`.
2. Delete `scripts/copy-local-registry.ts`, `scripts/generate-registry.ts`.
3. Delete `cli.json`.
4. Delete root `next.config.ts`, `tsconfig.json`, `components.json`, `source.config.ts`, `postcss.config.mjs`.
5. Update root `README.md` for monorepo layout.
6. Update `.github/` workflows: build command becomes `turbo run build`.
7. Verify Vercel: root `apps/docs`, install `bun install` at repo root, build `cd ../.. && turbo run build --filter=@repo/docs`.
8. Final smoke test against Vercel preview deploy.

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| Breaking user installs (URL/name/target change) | `generate.ts` constrained to produce identical `name`/`target` as today. Snapshot diff against pre-migration `registry.json` on every batch |
| fumadocs `dir` array not resolving | Validated via context7 (`dir: string[] \| undefined` confirmed). Pilot tests this end-to-end |
| Turbopack not resolving `@repo/registry/...` | Standard turborepo + bun workspaces pattern; used in create-faster |
| Bun catalogs incompatibility | Catalogs are bun resolver-level; downstream tools see flat installed deps. Transparent |
| Stub drift (forgotten regeneration) | Single entry `generate.ts` runs on `predev`/`prebuild` via turbo deps. CI runs `turbo run registry:generate --dry-run` and fails on drift |
| Vercel build dir wrong | Test on preview deploy before merging; `outputFileTracingIncludes` updated to `../../packages/registry/items/**/*` |

## Out of scope

- Adding a test runner.
- Renaming any registry item.
- Publishing `@repo/ui` to npm.
- Optimizing `__index__.ts` size.
- Splitting `apps/docs` into separate doc + playground apps.
